NexStore API Docs

Documentação completa das rotas da aplicação NexStore — e-commerce Full-Stack construído com Node.js, Express e PostgreSQL.

Node.js + Express 5 PostgreSQL 16 JWT + Bcrypt

Base URL: http://localhost:3000
As rotas retornam HTML renderizado pelo EJS (não JSON), pois é uma aplicação MVC Server-Side Rendering (SSR).
Autenticação via cookie HttpOnly contendo JWT — enviado automaticamente pelo navegador.

🔐 Autenticação

Rotas públicas de registro, login e logout. O JWT é armazenado em cookie HttpOnly após login.

GET /auth/login Exibe o formulário de login
200 OK
Renderiza a página de login
302 Redirect
Se já logado, redireciona para /
POST /auth/login Autentica o usuário e emite JWT
Body (application/x-www-form-urlencoded)
CampoTipoObrig.Descrição
emailstring✓E-mail cadastrado
senhastring✓Senha do usuário
302 → /
Login bem-sucedido; cookie JWT definido
302 → /auth/login
Credenciais inválidas ou conta inativa
POST /auth/register Cria novo usuário
Body (application/x-www-form-urlencoded)
CampoTipoObrig.Descrição
nomestring✓Nome completo
emailstring✓E-mail único
senhastring✓Senha (mínimo 8 caracteres)
cpfstring (11 dígitos)✓CPF sem formatação
data_nascimentodate (YYYY-MM-DD)✓Data de nascimento
telefonestringopcionalEx: 11999999999
302 → /
Cadastro realizado, já autenticado
302 → /auth/register
E-mail ou CPF já cadastrado
GET /auth/logout Encerra a sessão do usuário
302 → /auth/login
Cookie JWT expirado, sessão encerrada

Body (application/x-www-form-urlencoded)

Usuário não autenticado
302 → /carrinho
Carrinho vazio
POST /checkout Confirmar pedido e gerar recibo
Body (application/x-www-form-urlencoded)
CampoTipoObrig.Descrição
endereco_idinteger✓ID do endereço de entrega

Pedido criado, estoque baixado via trigger

📊 Admin — Dashboard
🔒 Requer is_admin = true no JWT. Acesso bloqueado para usuários comuns.
GET /admin/dashboard Faturamento, vendas e alertas de estoque

Exibe métricas analíticas: total de faturamento, número de pedidos por status e produtos com estoque abaixo do mínimo.

200 OK
Dashboard analítico renderizado
302 → /auth/login
Não é admin

GET /admin/pedidos Listar todos os pedidos
200 OK

Body (application/x-www-form-urlencoded)
CampoTipoValores aceitos

⚠️ Se o status for alterado para cancelado, um trigger PostgreSQL devolve automaticamente o estoque dos itens.

302 → /admin/pedidos
